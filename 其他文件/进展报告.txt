### 问题2：失负荷概率和EENS硬编码 ⏳ 待修复

**症状**：
```python
prob_load_shedding = 0.02 if avg_eens > 0.1 else 0.02  # 永远是0.02
expected_eens = 0.15  # 硬编码
```

**根本原因**：calculate_metrics()中全是硬编码假设，不是真实评估

**修复方案**：实现蒙特卡洛评估
- 生成1000个随机风电场景
- 对每个场景评估失负荷、弃风等
- 计算真实的概率和期望值

**预期时间**：1-2小时  
**关键文件**：蒙特卡洛评估框架.py（已准备）

### 问题3：风险模型被简化 ⏳ 待修复  

**症状**：
```python
# 直接用备用约束，完全跳过了论文的风险建模
min_reserve = 3.0 * wind_std
model += total_R_up >= min_reserve  # ✗ 这不是论文的方法
```

**正确做法**（论文公式）：
```python
# CCV分段线性化约束：
model += R_ens[t] >= a_params[k] * total_R_up + b_params[k]
# 这才是让RUC和DUC有成本差异的关键
```

**修复方案**：
1. 修复linearize_eens()中的参数计算（当前有符号符错误）
2. 恢复被注释掉的EENS_Risk约束
3. 让RUC通过风险建模获得比DUC更低的成本

**预期时间**：2-3小时  
**关键问题**：这是论文的核心差异，不修复RUC成本永远等于DUC
